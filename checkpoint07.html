Preguntas de Programación en JavaScript

1. ¿Qué diferencia a JavaScript de otros lenguajes de programación?

JavaScript es un lenguaje de programación versatil e interpretado, que se utiliza principalmente en el desarrollo web. Algunas de las características que lo diferencian de otros lenguajes son:

  • Entorno de ejecución: JavaScript se ejecuta directamente en el navegador web del usuario, sin necesidad de compilación previa.

  • Interpretación vs. Compilación: Es un lenguaje interpretado, lo que significa que el código se lee y ejecuta línea a línea por el navegador, en lugar de compilarse a un archivo ejecutable.

  • Tipado: JavaScript tiene un tipado dinámico, lo que significa que no es necesario declarar el tipo de dato de las variables antes de usarlas.

  • Modelo de ejecución: JavaScript funciona con un modelo basado en eventos, lo que le permite manejar interacciones con el usuario y tareas asícronas de manera eficiente.

  • Herencia: JavaScript utiliza un sistema de herencia basado en prototipos, en lugar de clases como en otros lenguajes.

2. ¿Cuáles son algunos tipos de datos en JavaScript?

En JavaScript, existen diferentes tipos de datos para representar información. Cada tipo de dato tiene características y usos específicos. Los tipos de datos básicos en JavaScript son:

  • Number: Representa tanto números enteros como de punto flotante. Ejemplo: 42, 3.14

  • String: Representa secuencias de caracteres. Ejemplo: "Hola, mundo"

  • Boolean: Representa valores lógicos true o false.

  • Object: Colección de propiedades. Ejemplo: { nombre: "Juan", edad: 30 }

  • Array: Colección ordenada de elementos. Ejemplo: [1, 2, 3, 4]

  • undefined: Representa una variable que ha sido declarada pero no asignada.

  • null: Representa la ausencia intencional de cualquier valor de objeto.

3. ¿Cuáles son las tres funciones de String en JavaScript?

JavaScript proporciona una gran variedad de métodos para manipular cadenas de texto (strings). Algunas funciones comunes:

  • length: Devuelve la longitud de la cadena. Ejemplo: "Hola".length devuelve 4.

  • split(separator, limit): Divide una cadena en un array de subcadenas, utilizando un delimitador específico. Ejemplo: "Hola, mundo".split(", ") devuelve ['Hola', 'mundo'].

  • toUpperCase(): Convierte todos los caracteres de la cadena a mayúsculas. Ejemplo: "hola".toUpperCase() devuelve "HOLA".

  • substring(start, end): Devuelve una parte de la cadena entre los índices start y end (sin incluir end). Ejemplo: "Hola, mundo".substring(0, 4) devuelve "Hola".

4. ¿Qué es un condicional?

Un condicional es una estructura de control que permite ejecutar un bloque de código solo si se cumple una cierta condición. En JavaScript, se usa comúnmente la sentencia if para crear condicionales.

let edad = 18;
    if (edad >= 18) {
        console.log("Eres mayor de edad");
    } else {
        console.log("Eres menor de edad");
    }

Además de la sentencia if, JavaScript ofrece otras formas de condicionales para manejar la lógica de control de flujo:

Sentencia if...else if...else

Permite evaluar múltiples condiciones de manera secuencial hasta que una de ellas sea verdadera. Si ninguna condición es verdadera, se ejecuta el bloque de código dentro del else final.

let nota = 85;
    if (nota >= 90) {
        console.log("Excelente");
    } else if (nota >= 70) {
        console.log("Bueno");
    } else if (nota >= 50) {
        console.log("Aprobado");
    } else {
        console.log("Reprobado");
    }

Operador ternario

Es una forma abreviada de la sentencia if...else que permite ejecutar una de dos expresiones basadas en una condición. Se utiliza el signo de interrogación (?) para separar la condición de las expresiones, y los dos puntos (:) para separar las dos expresiones.

let edad = 18;
    let mensaje = (edad >= 18) ? "Eres mayor de edad" : "Eres menor de edad";
    console.log(mensaje);

Sentencia switch

Permite seleccionar uno de varios bloques de código a ejecutar. Se evalúa una expresión una vez, y el valor de la expresión se compara con los valores de cada caso en la estructura switch. Si hay una coincidencia, se ejecuta el bloque de código correspondiente.

let fruta = "manzana";
    switch (fruta) {
        case "manzana":
            console.log("Es una manzana");
            break;
        case "naranja":
            console.log("Es una naranja");
            break;
        case "plátano":
            console.log("Es un plátano");
            break;
        default:
            console.log("Fruta desconocida");
    }

Estos diferentes tipos de condicionales permiten manejar de manera efectiva la lógica en los programas de JavaScript, asegurando que se ejecuten las acciones correctas en función de las condiciones evaluadas.

5. ¿Qué es un operador ternario?

Volviendo a la respuesta anterior, se puede definir el operador ternario como una forma concisa de escribir un condicional que asigna un valor basado en una condición. Es especialmente útil para simplificar estructuras if...else cortas. Tiene la siguiente sintaxis: condición ? valorSiVerdadero : valorSiFalso.

let edad = 18;
    let mensaje = edad >= 18 ? "Eres mayor de edad" : "Eres menor de edad";
    console.log(mensaje);

En el ejemplo anterior, si la condición edad >= 18 es verdadera, se asigna "Eres mayor de edad" a mensaje. Si es falsa, se asigna "Eres menor de edad" a mensaje.

Otro ejemplo de operador ternario

Considera un escenario en el que necesitas verificar si un número es par o impar:

let numero = 5;
    let tipo = (numero % 2 === 0) ? "Par" : "Impar";
    console.log(tipo);

En este ejemplo, si la condición numero % 2 === 0 es verdadera, se asigna "Par" a tipo. Si es falsa, se asigna "Impar" a tipo.

6. ¿Cuál es la diferencia entre una declaración de función y una expresión de función?

Tanto las declaraciones de función como las expresiones de función son mecanismos en JavaScript para crear bloques de código reutilizables. Sin embargo, existen diferencias sutiles en su funcionamiento y comportamiento que las hacen adecuadas para diferentes propósitos.

  1. Sintaxis:
    • Declaraciones de función:
      • Utilizan la palabra clave function seguida del nombre de la función, paréntesis () y llaves {} para el bloque de código.
      • Ejemplo:function saludar () {console.log("¡Hola!");}
    • Expresiónes de función:
      • Se definen utilizando la palabra clave const/let seguida de un igual (=), paréntesis () y llaves {} para el bloque de código.
      • Ejemplo: function saludar () { console.log("¡Hola!"); }
  2. Hoisting (Elevación):
    • Declaraciones de función:
      • Son elevadas (hoisted) al inicio de su ámbito léxico durante la fase de compilación.
      • Esto permite llamarlas antes de su definición en el código.
      • Ejemplo: saludar (); function saludar() {console.log ("¡Hola!"); }
    • Expresiones de función:
      • No son elevadas.
      • Deben definirse antes de ser llamadas.
      • Ejemplo: const saludar = function() { console.log ("¡Hola!"); }; saludar ();
  3. Ámbito:
    • Declaraciones de función:
      • Tienen un ámbito de función.
      • Las variables declaradas dentro de la función solo son accesibles dentro de la misma.
    • Expresiones de función
      • Tienen un ámbito de bloque.
      • Las variables declaradas dentro de la función son accesibles dentro del bloque en que se define la función.
  4. Uso:
    • Declaraciones de función:
      • Se utilizan principalmente para definir funciones reutilizables que forman parte de la estructura principal del programa.
      • Son útiles cuando se necesita llamar a la función antes de su definición.
    • Expresiones de función:
      • Se utilizan para crear funciones anónimas o para asignar funciones a variables o constantes.
      • Son útiles para encapsular código corto y específico o para pasarlo como argumuentos a otras funciones

Declaración de función con parámetros:

function saludar(nombre) {
        console.log("Hola", nombre, "!");
    }
    saludar("Juan");  // Imprime: ¡Hola, Juan!

Expresión de función como argumento:

const suma= function(a, b) {
            return a+b;
    };
    const resultado = suma(5,3); // resultado = 8

Es recomendable utilizar declaraciones de función para definir las funciones principales del programa y las expresiones de función para funciones anónimas, encapsular código corto o pasarlo como argumentos.

7. ¿Qué es la palabra clave "this" en JavaScript?

La palabra clave this en JavaScript se refiere al objeto en el que se encuentra actualmente. Su valor depende del contexto de ejecución en el que se utiliza:

  • En un método de objeto cuando this se encuentra dentro de un método de un objeto, se refiere al objeto al que pertenece ese método.
    • const objeto = {
                nombre: "JavaScript",
                mostrarNombre: function() {
                    console.log(this.nombre);
                    }
                };
                objeto.mostrarNombre();  // "JavaScript"

    Aqui, this dentro de mostrarNombre se refiere al objeto objeto, por lo que accede correctamente al valor de la propiedad nombre.

  • Si this se encuentra en una función simple (no asociada a un objeto), se refiere al objeto global ( en navegadores, generalmente el objeto window). Por ejemplo:
    • function mostrarNombreGlobal() {
                console.log(this.nombre)
                }
                mostrarNombreGlobal(); // Imprime el valor global de "nombre" 
                (si está definido)
  • En el contexto de un evento (como un clic de botón ), this se refiere al elemento HTML que recibe el evento. Por ejemplo:
    • <button id="miBoton">Haz clic</button>
                <script>
                document.getElementById("miBoton").addEventListener("click", function() {
                console.log(this.id); // Imprime "miBoton"
                });
            </script>

En resumen, el valor de this depende del contexto en el que se encuentra y puede variar según cómo se llame la función.
